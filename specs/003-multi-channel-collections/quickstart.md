# Quick Start Guide: Multi-Channel Collections

**Goal**: Organize multiple YouTube channel archives into a browsable collection in 10 minutes

This guide assumes you already have annextube installed and have archived at least one channel (see [001 quickstart](../001-youtube-backup/quickstart.md)).

## Prerequisites

- **annextube** installed and working
- **DataLad** installed (`pip install datalad` or `conda install -c conda-forge datalad`)
- At least one YouTube channel you want to archive

**Installation check**:
```bash
annextube --version
datalad --version
```

---

## Step 1: Create a Collection

A collection is a DataLad superdataset that holds multiple channel archives as subdatasets.

```bash
datalad create my-collection
cd my-collection
```

**What this does**:
- Creates a new DataLad dataset (git + git-annex initialized)
- This will be the parent container for all your channel archives

---

## Step 2: Add Channel Archives

### Option A: Create new channel archives (manual workflow)

For each channel you want to archive:

```bash
# Create a subdataset for the channel
datalad create -d . AnnexTubeTesting

# Initialize and backup the channel
cd AnnexTubeTesting
annextube init https://www.youtube.com/@AnnexTubeTesting
annextube backup
annextube export --channel-json
cd ..

# Save the new subdataset at the collection level
datalad save -m "Add @AnnexTubeTesting channel"
```

Repeat for additional channels:

```bash
datalad create -d . datalad
cd datalad
annextube init https://www.youtube.com/@datalad
annextube backup
annextube export --channel-json
cd ..
datalad save -m "Add @datalad channel"
```

### Option B: Import an existing external archive

If someone has published a channel archive (e.g., on GitHub, GitLab, or a
DataLad sibling), you can add it to your collection without re-downloading
all the content:

```bash
# Clone the published archive as a subdataset
datalad clone -d . https://github.com/user/ch-example

# Save at the collection level
datalad save -m "Add existing @example channel"

# Regenerate the collection summary
annextube aggregate --force
```

**How it works**:
- `datalad clone -d .` registers the archive as a subdataset of the
  collection (recorded in `.gitmodules`)
- The clone contains all git-tracked metadata (channel.json, videos.tsv)
  but annexes (video files) are not downloaded until you `datalad get` them
- Running `aggregate` discovers the cloned archive via its `channel.json`
  and includes it in `channels.tsv`

**Requirements for the cloned archive**:
- Must have a `channel.json` in its root (generated by `annextube export --channel-json`)
- Must have `.annextube/config.toml` to be recognized by `discover_subdatasets`
- Should have `videos/videos.tsv` for archive statistics

**After cloning**, you can update the archive alongside locally created
channels:
```bash
annextube collection backup --save
```

### Option C: Use `collection add` (recommended)

The simplest way to add a new channel:

```bash
annextube collection add https://www.youtube.com/@AnnexTubeTesting
annextube collection add https://www.youtube.com/@datalad
```

This automatically creates the subdataset, initializes, backs up, generates
`channel.json`, and saves at the collection level.

---

## Step 3: Generate the Collection Summary

Run the `aggregate` command to discover all channels and create `channels.tsv`:

```bash
annextube aggregate --force
```

**Output**:
```
Discovering channels in . (depth: 1)
Found 2 channel(s)
Generated channels.tsv with 2 channel(s)

Channels discovered:
  - AnnexTube Testing (AnnexTubeTesting): 5 videos
  - DataLad (datalad): 156 videos
```

**What this does**:
- Scans subdirectories for `channel.json` files
- Reads channel metadata and computes archive statistics from each channel's `videos.tsv`
- Generates `channels.tsv` at the collection root

If your channels are nested (e.g., `org/channel/`), use `--depth 2`:
```bash
annextube aggregate --depth 2 --force
```

---

## Step 4: Generate the Web Interface

```bash
annextube generate-web
```

**Open in browser**:
```bash
# Via built-in server (recommended for multi-channel)
annextube serve

# Or directly (single-channel only for file:// protocol)
xdg-open web/index.html
```

The web UI automatically detects multi-channel mode when `channels.tsv` is present:
- **Channel overview page**: Shows all channels with titles, statistics, video counts
- **Channel drill-down**: Click a channel to see its video listing
- **Breadcrumb navigation**: Navigate back to the overview from any channel

---

## Step 5: Update All Channels

### Manual update (current)

```bash
# Update each channel individually
cd AnnexTubeTesting && annextube backup && cd ..
cd datalad && annextube backup && cd ..

# Regenerate collection summary
annextube aggregate --force

# Save at collection level
datalad save -m "Regular update"
```

### Batch update (recommended)

```bash
annextube collection backup --save
```

This updates all channels sequentially, reports per-channel results, and saves changes at the collection level.

For automated updates, add to crontab:
```bash
# Update collection daily at 2 AM
0 2 * * * cd /path/to/my-collection && annextube collection backup --save --push
```

## Result Structure

After completing the steps above, your collection looks like this:

```
my-collection/                    # DataLad superdataset
├── .git/
├── .gitmodules                   # Managed by DataLad
├── channels.tsv                  # Collection summary (generated by aggregate)
├── AnnexTubeTesting/             # DataLad subdataset
│   ├── .annextube/config.toml
│   ├── .git/
│   ├── channel.json              # Channel metadata (generated by export)
│   ├── videos/
│   │   ├── videos.tsv
│   │   └── 2026/01/...
│   └── playlists/
│       └── playlists.tsv
├── datalad/                      # DataLad subdataset
│   ├── .annextube/config.toml
│   ├── .git/
│   ├── channel.json
│   ├── videos/videos.tsv
│   └── ...
└── web/                          # Generated web UI
    └── index.html
```

Each channel archive is a complete, independent DataLad dataset that can be:
- Used standalone (copy out of the collection and it still works)
- Pushed to its own remote (`cd datalad && datalad push`)
- Shared with others (`datalad clone <url>`)

## Common Tasks

### Check collection status
```bash
datalad status         # See changes across collection
datalad subdatasets    # List all channel subdatasets
```

### Add another channel later
```bash
datalad create -d . new-channel
cd new-channel
annextube init https://www.youtube.com/@NewChannel
annextube backup
annextube export --channel-json
cd ..
datalad save -m "Add @NewChannel"
annextube aggregate --force
```

### View collection summary
```bash
# View in terminal
column -t -s $'\t' channels.tsv

# View with VisiData (if installed)
vd channels.tsv
```
