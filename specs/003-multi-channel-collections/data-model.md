# Data Model: Multi-Channel Collections

**Feature**: 003-multi-channel-collections
**Date**: 2026-03-21
**Purpose**: Define data structures for multi-channel collection entities

## Overview

This data model extends the 001-youtube-backup data model with collection-level entities. All data is stored in **file-based formats** (TSV, JSON, TOML) with **no database dependencies** (Constitution Principle XI: Storage Simplicity).

**Storage Strategy**:
- **Collection summary**: `channels.tsv` at collection root (one row per channel)
- **Per-channel metadata**: `channel.json` in each channel directory
- **Per-channel content**: Existing `videos.tsv`, `playlists.tsv`, video metadata JSON (unchanged from 001)
- **Collection config**: `[collection]` section in `.annextube/config.toml`

## Entities

### 1. channels.tsv (Collection Root)

Top-level summary of all channels in the collection. Generated by the `aggregate` command by scanning for `channel.json` files.

**Location**: `channels.tsv` at collection root directory

**Columns**:

| Column | Type | Description |
|--------|------|-------------|
| `channel_id` | string | YouTube channel ID (e.g., `UCZXGJjgg0cB5vqd5aVcJ0pQ`) |
| `title` | string | Channel display name |
| `custom_url` | string | Channel handle (e.g., `@apopyk`) |
| `description` | string | Channel description (TSV-escaped) |
| `subscriber_count` | integer | Subscriber count at last fetch |
| `video_count` | integer | Total public videos reported by YouTube |
| `playlist_count` | integer | Number of playlists |
| `total_videos_archived` | integer | Actual videos archived locally (from videos.tsv row count) |
| `first_video_date` | string | Earliest video publish date in archive (ISO 8601 date) |
| `last_video_date` | string | Latest video publish date in archive (ISO 8601 date) |
| `last_sync` | string | Last sync timestamp (ISO 8601 datetime, deprecated -- recoverable from git history) |
| `channel_dir` | string | Relative path from collection root to channel directory |

**Example**:
```tsv
channel_id	title	custom_url	description	subscriber_count	video_count	playlist_count	total_videos_archived	first_video_date	last_video_date	last_sync	channel_dir
UCZXGJjgg0cB5vqd5aVcJ0pQ	Andriy Popyk	@apopyk	Ukrainian programming tutorials	15234	287	12	287	2018-03-15	2026-01-20		ch-apopyk
UC1234567890ABCDEF	DataLad	@datalad	Data management tools	8543	156	5	156	2019-06-01	2025-12-15		ch-datalad
```

**Validation**:
- `channel_id`: Required, non-empty
- `title`: Required, non-empty
- `total_videos_archived`: Non-negative integer, computed from local `videos.tsv`
- `channel_dir`: Required, must be a valid relative path
- Rows sorted by `title` (case-insensitive)

**Generation**: Produced by `annextube aggregate` command. See `annextube/cli/aggregate.py` for implementation.

### 2. channel.json (Per-Channel)

Detailed metadata for a single channel archive. Used as the discovery marker by the `aggregate` command -- a directory containing `channel.json` is recognized as a channel archive.

**Location**: `channel.json` at root of each channel archive directory

**Schema**:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `channel_id` | string | yes | YouTube channel ID |
| `name` | string | yes | Channel display name |
| `description` | string | yes | Channel description |
| `custom_url` | string | no | Channel handle (e.g., `@apopyk`) |
| `subscriber_count` | integer | yes | Subscriber count at last fetch |
| `video_count` | integer | yes | Total public videos reported by YouTube |
| `avatar_url` | string | no | Channel avatar thumbnail URL |
| `banner_url` | string | no | Channel banner image URL |
| `country` | string | no | Channel country |
| `videos` | string[] | yes | List of video IDs in channel |
| `playlists` | string[] | yes | List of playlist IDs in channel |
| `created_at` | string | no | Channel creation date (ISO 8601) |
| `archive_stats` | object | no | Computed statistics from local archive |

**`archive_stats` sub-object**:

| Field | Type | Description |
|-------|------|-------------|
| `total_videos_archived` | integer | Row count from local `videos.tsv` |
| `first_video_date` | string | Min `published_at` from `videos.tsv` (ISO 8601 date) |
| `last_video_date` | string | Max `published_at` from `videos.tsv` (ISO 8601 date) |
| `total_duration_seconds` | integer | Sum of all video durations |
| `total_size_bytes` | integer | Sum of all video file sizes |

**Example**:
```json
{
  "channel_id": "UCZXGJjgg0cB5vqd5aVcJ0pQ",
  "name": "Andriy Popyk",
  "description": "Ukrainian programming tutorials and tech talks",
  "custom_url": "@apopyk",
  "subscriber_count": 15234,
  "video_count": 287,
  "avatar_url": "https://yt3.ggpht.com/...",
  "country": "Ukraine",
  "created_at": "2018-03-15T00:00:00Z",
  "videos": ["dQw4w9WgXcQ", "abc123"],
  "playlists": ["PLxxxxxx", "PLyyyyyy"],
  "archive_stats": {
    "total_videos_archived": 287,
    "first_video_date": "2018-03-15",
    "last_video_date": "2026-01-20",
    "total_duration_seconds": 145230,
    "total_size_bytes": 25600000000
  }
}
```

**Note**: The `last_sync` and `fetched_at` fields were deprecated in the Channel model (see 001 data-model.md). Sync time is recoverable from git history.

**Generation**: Produced by `annextube export --channel-json`. See `annextube/services/export.py` for implementation.

### 3. Collection Configuration

The `[collection]` section in `.annextube/config.toml` at the collection root defines defaults for managing channels.

**Location**: `.annextube/config.toml` at collection root

**Schema**:

```toml
[collection]
# Default init options applied to `collection add`
comments_depth = 0          # int: Default comment depth for new channels
curation = true             # bool: Enable caption curation by default
search = true               # bool: Enable search index by default
include_playlists = "all"   # str: "all", "none", or regex pattern
include_podcasts = "none"   # str: "all", "none", or regex pattern

# Common config to embed into each new channel on `collection add`
common_config = ".annextube/common-config.toml"  # str or null

# Push settings for `collection backup --push`
push_remote = "datalad-public"  # str or null
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `comments_depth` | integer | `0` | Default comment reply depth for new channels |
| `curation` | boolean | `true` | Enable caption curation for new channels |
| `search` | boolean | `false` | Enable search index for new channels |
| `include_playlists` | string | `"none"` | Playlist discovery mode for new channels |
| `include_podcasts` | string | `"none"` | Podcast discovery mode for new channels |
| `common_config` | string or null | `null` | Path to common config file to embed |
| `push_remote` | string or null | `null` | Remote name for `collection backup --push` |

**Behavior**:
- These values are **copied** into new channel configs at `collection add` time
- Per-channel configuration always takes precedence after creation
- The `common_config` file is embedded (copied) into each new channel, not referenced

**Key relationships**:
- `channels.tsv` is derived from all `*/channel.json` files (via `aggregate`)
- Each `channel.json`'s `archive_stats` are derived from that channel's `videos/videos.tsv`
- The web UI uses `channels.tsv` to render the overview, then loads `{channel_dir}/videos/videos.tsv` on demand
- Each channel subdataset is independently usable as a standalone archive
- Collection config defaults are a one-time copy, not a live reference
